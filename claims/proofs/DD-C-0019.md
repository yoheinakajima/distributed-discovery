# DD-C-0019 — private-team benchmark inequalities

## Claim

In the declared DD-001 model, direct clue-following value is at most the private-team frontier. If a pooled planner observes all private signals and can emulate every private-team action profile under the same physical budget and objective, the private-team frontier is at most the pooled planner frontier.

## Proof

Direct clue-following is the deterministic feasible profile \(f_i(x)=x\) for every agent, so maximizing over all feasible private-team profiles cannot yield less.

For the upper inequality, fix any private-team profile \(f\). After observing the full signal vector, the pooled planner can choose the same action vector \((f_1(X_1),\ldots,f_N(X_N))\). The emulated protocol has the identical outcome distribution and discovery value. Therefore every private-team value is feasible in the pooled-planner class; maximizing over the latter weakly dominates maximizing over the former.

The upper inequality requires the stated emulation and matched physical constraints. It is not asserted when pooling changes compliance, rewards, privacy feasibility, timing, or the social objective.

## Computational check

The primary 21-point exact grid checks both inequalities for every case. Computation illustrates but is not the proof.
