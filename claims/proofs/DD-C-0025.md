# DD-C-0025 — canonical signature state-space count

## Feasible labeled signatures

Fix the set of $n_0$ nonfixed targets. Residual capacities $r_t=c_t-d_t$ are weak compositions of $n_0$ into $M$ labeled columns, giving $\binom{n_0+M-1}{M-1}$ candidates. The fixed-column bound $r_t\le n_0$ is automatic. For each of the $n_0$ nonfixed columns, exactly one composition violates $r_t\le n_0-1$: all $n_0$ residual units lie in that column. These invalid compositions are distinct. Choosing the nonfixed set and summing gives

\[
S_M=\sum_{n_0=0}^{M}\binom{M}{n_0}
\left[\binom{n_0+M-1}{M-1}-n_0\right].
\]

At $M=16$, the exact result recorded by the primary run is $S_{16}=148{,}348{,}284{,}928$.

## Target orbits and agent multisets

For one signature, target relabeling preserves and is characterized by two multisets: residual counts on fixed targets and residual counts on nonfixed targets. The implementation counts their bounded integer partitions and obtains 5,806 individual-signature orbits. Exhaustive enumeration agrees with this partition counter for every $M\le5$ in the audit.

Agent permutation is lossless for the common objective, so without a target quotient the eight-agent signature-multiset count is

\[
\binom{S_{16}+8-1}{8}.
\]

The exact expanded integer is stored in the claim and run. Independently sorting each agent's signature is not lossless because discovery depends on how their target coordinates align. Therefore 5,806 is not substituted into the agent-multiset formula.

## Barrier scope

The run's minimum-memory figure assigns only sixteen 64-bit words to each multiset state, so it is a lower bound on a naive retained failure-vector table, not a hardware prediction. The result rules out that declared enumeration plan; it is not a complexity lower bound for all algorithms and supplies no canonical objective upper bound.
