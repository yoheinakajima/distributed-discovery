# DiscoveryBench Agents v1 registration gate

This prospectus authorizes registration preparation only. It creates no study
ID, run, provider call, model download, agent execution, cost, trace, result,
private seed, or holdout.

Central question: do real software-agent teams exhibit duplication, source
concentration, shared-clue over-attention, consensus collapse, and
portfolio-recovery failures predicted by exact Distributed Discovery tasks?

Candidate task families are common-source acquisition; one-reader versus
broadcast; point versus shortlist signals; consensus collapse versus recovery;
and threshold teams. Registration must freeze the benchmark and generator,
provider/model identifiers and snapshots, system and user prompts, agent
count/identity, topology, message budget, memory, tools, temperature, top-p,
seed, retries, stopping, context, cost cap, timeout, raw-trace policy, and
safety boundary.

Contamination control requires automated cryptographic custody without a human
custodian. The generator is committed before evaluation, provider/model
versions freeze before holdout generation, and a cryptographically secure
random-number generator creates each seed automatically. The runner publishes
a seed-commitment hash, encrypts holdouts and answer keys, logs access, and
isolates evaluated agents from generator internals, seeds, and answers.
Outputs become immutable before unsealing; post-evaluation verification must
match the commitments. Multiple independent batches are required. Generator
parameters, seeds, and answers are released later when safe.

The contamination red line detects exact or near-verbatim public benchmark
values, wording, task IDs, or solution patterns in evaluated outputs. Public
tasks are calibration only.

At least two genuinely different model families and a feasible local/open
baseline are required. Metrics are group discovery, distinct coverage,
duplication, planner/private regret, recovery attainment, source diversity,
communication compression, distance from best/worst registered equilibrium,
invalids, compliance, and operational cost/tokens. There is no universal
ranking or default composite score.

Boundaries: software agents are not humans; results would be provider,
snapshot, prompt, topology, and sampling specific; no organizational
generalization, leaderboard claim, or provider endorsement follows.

Stop if contamination cannot be bounded, versions cannot be frozen, cost is
unauthorized, safe trace preservation is impossible, leakage is unresolved,
reasoning cannot be separated from memorization, different model families are
unavailable, or the design reduces to generic accuracy.
