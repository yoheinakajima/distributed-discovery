# DiscoveryBench Agents v1 registration decision

Decision: `register-instrument-under-dd010`.

DiscoveryBench Agents v1 is an offline, registered-not-executed instrument
layer owned by DD-010. It versions agent protocol, generator, prompt, trace,
custody, contamination, analysis, and result contracts independently of the
unchanged DiscoveryBench v1/v2/v3 content versions. It is not DiscoveryBench
v4.

No study ID or claim ID is allocated. No run, evidence, trace, private seed,
holdout, answer key, model call, model download, or cost exists. The four
provider/model records are candidate audits, not selections or endorsements.
Future execution requires a separate issue and branch, exact snapshot and
retention verification, explicit owner cost authorization, an immutable
evidence package, and both verification paths.

The gate passes because the scientific ownership is explicit; five task
families and five agent architecture contrasts are finite and machine-gradable;
two fixed cloud model families and a revision-pinned open candidate are
available for later consideration; the estimated base campaign fits the
non-authorizing $750 recommendation; and custody, contamination, trace,
statistical, metric, verification, and 24-corruption contracts have credible
offline validators.

The instrument stops before execution on an ownership/version mismatch,
custody or contamination breach, moving/unfrozen model identifier, unsafe
trace, absent cost authorization, inadequate model diversity, generic benchmark
overlap, or verifier disagreement.

The exact next gate is **DiscoveryBench Agents v1 offline instrument
implementation**. Its exact next command is `git status --short --branch`; its
exact next file is `plans/DISCOVERYBENCH_AGENTS_V1_IMPLEMENTATION.md`. That file
is intentionally not created by this registration.
