# First compendium release plan

Candidate version: `0.1.0`. Candidate tag form:
`dd-compendium-v0.1.0`. Neither is an existing release or authorization.

Current public instrument identity: **TreasureBench**, with Treasure Hunt as
its interactive companion and DiscoveryBench retained only for historical and
frozen compatibility. A future release manifest must use the TreasureBench
display name while preserving every frozen identifier and historical path.
Before any package or DOI action, the owner must review the
TreasureBench namespace checklist and perform the one-time preliminary
USPTO/EUIPO register checks. Those checks are nonlegal signals, not clearance.

1. Resolve every owner decision in the licensing matrix.
2. Select a clean candidate commit after the active substantive pull request
   has merged.
3. Regenerate the evidence manifest with the immutable commit and UTC time.
4. Run all repository, paper, site, checksum, and release-readiness gates.
5. Complete the owner activation checklist.
6. Create the annotated tag and GitHub Release only with explicit owner
   authorization.
7. Verify Zenodo ingest, then record the real version and concept DOIs in a
   reviewed follow-up.

This plan does not authorize a `treasurebench` package, namespace reservation,
tag, release, Zenodo integration, DOI, or arXiv submission.

Recovery follows the immutable-version rule: never move a published tag or
silently overwrite a deposit; correct through a new version and durable
correction record.
