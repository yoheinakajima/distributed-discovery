# DiscoveryBench Agents v1 base allocation audit

The deterministic public registry contains 200 slots: 40 per family and four
50-slot batches with ten slots from every family in every batch. All 138
canonical generator cells appear at least once. The remaining 62 slots are
predeclared boundary-priority repeats. The four target/agent relabeling class
combinations each receive 50 slots.

The registry names generator cells and isomorphism classes only. It contains
no seed, private task, prompt, answer, holdout, key, or outcome. It is frozen
for a future base campaign but is not selected for the next pilot.
