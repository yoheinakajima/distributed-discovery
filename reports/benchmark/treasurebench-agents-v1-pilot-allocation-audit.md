# TreasureBench Agents v1 pilot allocation audit

The frozen public allocation contains 50 slots: ten for each of the five
registered task families. Each family includes two slots in each of the
strict, equality/tie, boundary, protocol-sensitive, and parser-sensitive
categories. Target and agent relabeling classes are balanced 25/25 across the
batch. Every slot is compatible with all five architectures and has an exact
comparator path.

The allocation contains no seed, task text, answer, hidden target, prompt, or
outcome-dependent choice. It does not reuse either public-calibration
instances or future base-campaign private instances; future base execution
must use new slot identity, seed domain, and material.
