# Context isolation and internal independence

A context-isolated reproduction receives only the registered question, model,
inputs, schemas, declared outputs, resource limits, and acceptance rules.
Unless explicitly required, it does not receive the producer's derivation,
source code, expected numerical answers, private scratch work, or conclusion.

The audit record must list every supplied artifact and checksum. Any accidental
context leak is recorded and the affected independence claim is withdrawn or
repeated under a new authorized record. Shared model/provider origin, shared
training data, and shared public sources are disclosed limitations.

Context isolation supports an internal robustness claim only. It is not
outside reproduction, independent scholarly validation, or peer review.
