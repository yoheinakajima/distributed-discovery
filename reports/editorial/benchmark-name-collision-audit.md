# Benchmark public-name collision audit

Audit date: 2026-07-23. This is a scholarly/software collision screen, not a
trademark opinion or legal clearance.

`DiscoveryBench` has a fatal collision with the established ICLR 2025
benchmark *DiscoveryBench: Towards Data-Driven Discovery with Large Language
Models*, its [official proceedings record](https://proceedings.iclr.cc/paper_files/paper/2025/hash/0d70af566e69f1dfb687791ecf955e28-Abstract-Conference.html),
[AllenAI GitHub repository](https://github.com/allenai/discoverybench), and its
use in the AstaBench ecosystem. Reusing that public name for the first external
scholarly benchmark artifact would create a likely false association.

Exact-name searches covered arXiv, official conference results, Crossref/DOI
search surfaces, DBLP, GitHub, Python package indexes, general web results,
benchmark-directory signals, and trademark-like web results. No established
exact-name result was found for ActionPortfolioBench, SearchPortfolioBench,
EvidenceToActionBench, DiscoveryArchitectureBench, PortfolioSearchBench,
ActionDiversityBench, CollectiveSearchBench, DiscoveryPolicyBench, or
ArchitectureEffectsBench.

Search limitations are material: indexing is incomplete and time-dependent;
near-name, acronym, unpublished, package, domain, and jurisdiction-specific
conflicts may be absent; absence from exact-name results is not ownership or
legal clearance. Every alternative therefore retains unresolved ambiguity.
The live search must be repeated before any external release.
