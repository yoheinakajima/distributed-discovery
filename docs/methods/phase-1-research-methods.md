# Phase 1 research methods record

This is a factual repository methods record, not a scientific paper or a claim
that process establishes research quality.

Work was organized through GitHub issues, one substantive branch and pull
request at a time, and living ExecPlans. Scientific statements were entered in
an authoritative claim ledger with explicit status, scope, source references,
and immutable run IDs where numerical. Registered studies froze models,
resource bounds, commands, seeds where applicable, expected outputs,
verification, and corruption tests before primary execution.

Passing run directories are immutable and include source commit, environment,
commands, timestamps, checksums, and validation. Exact computations,
independent reproductions, synthetic Monte Carlo estimates, sourced upstream
results, proofs, identities, counterexamples, bounded nulls, and failed
searches remain separately labeled. Independent implementations and deliberate
corruptions were used where recorded; neither is treated as external review.

Paper admission was a separate editorial gate. Paper builders resolved claim
IDs and source artifacts, generated provenance, built PDFs deterministically,
and required visual inspection. Site generation checked public-safe outputs,
routes, relations, downloads, accessibility, and no-tracking behavior.

Human owner decisions determined program scope, stops, publication posture,
and external actions. The current operating boundary describes an AI-powered
lab under human PI ownership and final gate authority. Agent assistance is
evidenced by plans, commits, and repository artifacts where present; this
record does not infer autonomy. Internal AI roles may produce, reproduce,
verify, attack, reduce, review literature, corrupt, audit editorial scope,
integrate, and accept within declared authority. No human study, external
review, peer review, or community validation is claimed.
