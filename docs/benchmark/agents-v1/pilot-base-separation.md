# Pilot/base separation

The sealed pilot is engineering evidence only. Its tasks, prompts, ciphertexts,
outputs, traces, and tuning decisions are excluded from base inference.

Pilot task material cannot be reused in base. Base batches are generated only
after final implementation, model, authorization, custody, trace, and
statistical freezes. Any post-pilot change to a prompt, generator, evaluator,
parser, architecture, metric, action schema, or trace policy quarantines the
entire pilot. The changed instrument requires a new sealed batch.

Pilot observations may identify operational failure but cannot tune the base
while preserving pilot evidence. Pilot and base results are never pooled.
